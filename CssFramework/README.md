# CSS Framework

 ## 서론
 CSS Framework는 웹 개발을 할 때 CSS 파일을 통으로 제공하여 쉽고 빠르게 개발할 수 있도록 도와준다. 이미 틀에다 채우기만 하면 완성되는 CSS 디자인들을 HTML Tag에 클래스로 선언만 하면 된다. 여기서 의아한 점은 CSS Framework가 생긴 이유다. 이미 CSS의 역사가 오래되어 인터넷에 돌아다니는 Legacy code나 library도 많은데 굳이 Framework까지 발전한 이유가 뭘까? 본 문서는 이 의문을 중점으로 내용을 전개해 나갈 것이다. 예시로 Bootstrap으로 다룰 것이며 해당 개발진들이 어떤 의도로 CSS Framework를 개발했는지 설명하겠다. 일단 반응형 UI에 대한 내용은 배제했다.

 ## 1. BootStrap이란?

 부트스트랩은 CSS Component(조립만 하면 되는 부품)를 제공하는 CSS Framework이다. 부트스트랩이 나오게 된 배경은 위키에 다음과 같이 명시되어 있다. “기존에 여러 인터페이스에 대응하기 위한 다양한 라이브러리들이 존재했지만, 정형화되지 않아 유지 보수에 어려움이 있었다.” 해당 상황을 좀 더 알아보기 위해 마크오토(트위터 개발 및 부트스트랩 제작자) 블로그에 있는 당시의 회고를 찾아 봤다.  내부적인 툴도 물론이고 형식적인 디자인에 대한 빠른 개발을 하지 못하는 상황을 문제로 인지한 후 해당 프레임워크의 필요성을 느꼈다고 한다. 그러면 어떤 식으로 개발 속도를 높였는지 기존 code와 비교하면서 살펴 보자.

## 결론

 위에 예시처럼 카드나 네비게이션 바 등 대다수의 웹 페이지에서 필수적으로 사용되는 스타일이 있다. 네비게이션 바 같은 경우는 화면에 고정시키거나 가로를 얼만큼 차지하게 할 지 등의 보일러 플레이트가 굉장히 많다. 그러나 해당 보일러 플레이트를 프레임워크로 획기적으로 줄일 수 있기 때문에 사용성이 높은 것으로 보인다. 보일러 플레이트의 감소는 유지 보수를 하기 쉬운 환경으로 만들어주기 때문에 유지 보수에 대한 부분도 좋아졌을 것이다. 다만 복잡한 HTML 구조에 쉽게 적용하긴 어려울 것으로 보이며 animation 등에 대한 처리도 성능적으로도, 개발할 때에도 불편하다고 한다. 그래서 간단한 UI 구성할 때 사용하는 것이 좋아 보이고, 말 그대로 기본적인 틀을 잡을 때 사용하고 세부적인 부분은 css코드로 덮어쓰는 방식을 사용하는 것이 가장 좋아보인다.  
 
 마지막으로 단순한 CSS Style에 Component를 기조로 하는 프레임워크 기반을 구성한 부분을 고평가하고 있다. component로 틀을 잡고 css 코드를 통해 세부적인 조절을 하는 디자인 시스템을 WEB 개발 환경으로 만들었다는 것이다. 디자인에서 component라는 개념을 리액트, 플러터, 안드로이드 등에서도 사용하는데 어떻게 보면 현시대에서 사용하는 디자인 시스템을 선도했다고 본다. 이러한 점이 부트스트랩의 등장으로 사라졌던 수많은 CSS 라이브러리이든,기존 프레임워크 등을 대체할 수 있었던 이유라고 생각한다.
